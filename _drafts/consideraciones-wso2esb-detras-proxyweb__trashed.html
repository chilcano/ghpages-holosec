WSO2 ESB es un motor de mediación basado en Apache Synapse que permite mediar (modificar, enrutar, transformar, etc.) mensajes entrantes, por lo general SOAP, REST, JSON, POX, etc., pero para invocar esta mediación, WSO2 ESB nos permite exponer dichas funcionalidades como "Servicios". En este caso, los servicios nos sirven de interfaz con los que nosotros, desde internet por ejemplo, podemos invocar usando un cliente SOAP, REST o el que fuere.

Dichas interfases (servicios) son abiertos, es decir, que cualquier puede invocarlos sabiendo su URL. Desde el punto de vista de Seguridad, estas interfases son siempre filtradas, o deberían serlo siempre. Desde el punto de vista técnico, se suele usar Apache HTTP server como filtro, es decir, restringe ciertas URL, modifica la URL del servicio hacia internet o manipula ciertos atributos de la cabecera del mensaje SOAP o HTTP con la intención de no proveer información de la Organización hacia fuera, información como:

  • Direcciones IP, Direcciones de Red y Subredes.
  • Nombres del host.

  • Nombres y versiones del software servidor, etc.

En esta entrada os explicaré cómo configurar Apache HTTP server como Proxy Web y WSO2 ESB para poder acceder correctamente a los servicios expuestos por éste.
